# Telegram bot for kayak tours

Бот допомагає створювати тури, показувати детальну інформацію з фото, приймати заявки на кілька учасників, давати кнопку оплати Monobank і надсилати нагадування за 3 дні та за 1 день.

## Запуск

1. Створіть бота через `@BotFather` і скопіюйте токен.
2. Скопіюйте `.env.example` у `.env`.
3. Заповніть:
   - `BOT_TOKEN` - токен Telegram-бота.
   - `ADMIN_IDS` - ваш Telegram ID. Якщо адмінів кілька, пишіть через кому.
   - `MONO_PAYMENT_URL` - посилання на вашу постійну банку Monobank.
   - `CAMPING_PAYMENT_URL` - окрема банка Monobank для бронювання кемпінгу.
4. Встановіть залежності:

```powershell
python -m pip install -r requirements.txt
```

5. Запустіть бота:

```powershell
python bot.py
```

## Керування ботом

Основний сценарій працює через нижнє меню біля поля введення повідомлення. Після `/start` бот показує кнопки:

- `🛶 Доступні тури` - список турів і перехід до деталей.
- `🎒 На 1 день` - готовий список речей для одноденного туру.
- `🏕 З ночівлею` - готовий список речей для туру з ночівлею.

Адмін додатково бачить кнопки:

- `➕ Створити тур` - покрокове створення туру: дата, час, ціна, передплата, кількість місць, формат, фото, опис і маршрут.
- `📅 Мої тури` - список активних турів.
- `📝 Заявки` - останні заявки з кнопками підтвердження оплати або скасування.

Дату й час туру вводьте у європейському форматі: `15.07.2026 09:00`.

Кнопка `Оплату отримано` потрібна адміну після ручної перевірки Monobank. Після натискання бот позначає заявку оплаченою і надсилає клієнту повідомлення, що його заявка активна.

Команди `/tours`, `/packing_day`, `/packing_overnight`, `/newtour`, `/mytours`, `/bookings`, `/paid ID`, `/cancel ID` залишені як запасний варіант, але користуватися ними не обов'язково.

## Оплата Monobank

Бот завжди використовує посилання `MONO_PAYMENT_URL` з `.env`, тому під час створення туру окремо вводити банку не потрібно.

Для кемпінгу бот використовує окреме посилання `CAMPING_PAYMENT_URL`.

У кемпінгу клієнт обирає конкретний номер спорядження: звичайні намети `1-6`, намети-гамаки `1-4`. Уже заброньовані номери бот не показує для повторного вибору.

Після бронювання кемпінгу клієнт отримує окрему кнопку з локацією кемпінгу на карті.

У картці туру є кнопка `👥 Учасники туру`. Гості бачать тільки імена зареєстрованих учасників, адміни бачать повну інформацію: контактну особу, телефон, Telegram і список учасників.

Повноцінна автоматична перевірка платежів через Monobank Acquiring потребує merchant-токена та підключення API. У цій версії бот веде людину на оплату, а адмін підтверджує оплату кнопкою в заявці.
